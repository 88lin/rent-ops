# 模式：risk — 风险检测

输入小区名，agent 搜集社交媒体上的真实租客反馈和避雷信息。

## 输入

用户输入：`/rent risk {小区名}`

如果未提供小区名，询问：

> "你想查哪个小区的避雷信息？"

## 搜集流程

### 1. 小红书避雷帖

WebSearch 以下 query（每个 query 独立搜索）：

- `"{小区名}" 避雷 site:xiaohongshu.com`
- `"{小区名}" 踩坑 site:xiaohongshu.com`
- `"{小区名}" 租房 site:xiaohongshu.com`

### 2. 豆瓣/知乎帖子

- `"{小区名}" 租房 避雷 site:douban.com`
- `"{小区名}" 租房 怎么样 site:zhihu.com`

### 3. 通用搜索

- `"{小区名}" 租房 投诉`
- `"{小区名}" 物业 差评`

### 4. 二房东检测

如果有该房源的发帖人信息（联系电话或昵称），搜索：
- `"{联系电话}" 租房` — 看同一号码是否在多个不同小区挂房
- `"{发帖人昵称}" 租房` — 看是否批量发帖

同一人在 3 个以上不同小区挂房源 → 高概率二房东。

### 5. 隔断线索

如果有房源面积信息，检查：
- 一居室面积 < 20㎡ → 疑似隔断
- 户型描述与面积不匹配（如 "两室" 但只有 30㎡）

## 输出格式

```
🔍 风险报告：{小区名}
━━━━━━━━━━━━━━━━━━━━━━━━━━

📱 小红书租客反馈（{N} 条）
  👍 正面：{摘要}
  ⚠️ 负面：{摘要}

💬 豆瓣/知乎评价（{N} 条）
  {摘要}

🚩 风险信号
  {如有二房东线索、隔断疑点、物业投诉等}

📊 综合风险等级：低 / 中 / 高
  {一句话总结}
```

## 注意事项

- 搜索结果可能有噪音，agent 需要判断相关性，过滤掉不相关的帖子
- 如果搜索不到任何结果，如实告知「未找到该小区的避雷信息，可能是较新的小区或帖子较少」
- 不要编造负面信息 — 没找到就是没找到
- 如果该小区已在 tracker 中，可以将风险信息更新到对应条目的备注中
